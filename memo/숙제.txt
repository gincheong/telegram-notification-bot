#. 로그 파일을 외부로 전송하기? 외부로 전송하는 다른 코드를 짤 것인가
아니면 애당초 외부에 파일을 쓸 것인가?
외부라고 하면 어디에 써야 할 것인지? 모바일 환경으로 볼 수 있어야 좋다.


***

V. 그룹채팅방에 start를 입력했을 때 안내 메세지
-> 개인 메세지로 보내지도록 함
-> 개인 메세지로 보내는 데 문제가 있을 것 같아서, 수정함
그리고 어차피 지금은 상황이 다름

V. 알림 메세지에 italic 표현
-> Bold처리함

V. 각종 예외처리 추가함

V. 디비 연결하는거 코드 수정, 클래스 만든거 써서

V. 항시 작동하는걸로 하되, 원할 때 사용자가 끌 수 있도록 한다. 
-> 그룹별로 알림을 ON/OFF 할 수 있게 했음

V. 서비스 도중에 부득이하게 서버를 내렸다가 켜야 하는 경우,
  유저가 /start를 다시 입력하지 않아도 작동하게 하는 방법
-> /start같은걸 입력하지 않아도 자동으로 작동하게 한다

V. 디비 구조를 바꾸고, 그에 맞춰 키워드 알림을 보내는 구조를 수정함

V. 영어 대소문자 구분이 되는 상태, 안 하도록 바꾸기
-> islower() 를 써서 소문자상태로 비교하도록 함

V. 명령어 수정하기, /list라고 하면 그룹 리스트인지, 키워드 리스트인지
직관적이지 않음, klist 라던지, kdelete라던지 구분을 하자

V. getMessage에서 알람 전송하는 부분 수정했음, 테스트해야함 근데 뭔지모르겠는데 잘 안됨
/gtoggle도 입력이안되네
-> getMessage, gtoggle 각각 다른 문제로 안 되던 것 수정

V. 날짜 바뀔 때 로그 파일 이름이 바뀌어야 한다
-> 임시로 해결은 함, 근데 더 나은 방법은 없을지

V. gtoggle 명령어 확장, 그룹 이름을 보여주기?
V. 등록한 그룹을 삭제하는 기능, /group으로 제어? 명령어 이름 바꾸거 나서
-> 실행할 때 목록을 또 보여주기도 함, glist 라는 명령어로 별개로 목록 보여주기만 하기도 추가

V. Bothub 써보기
-> Bothub 전용 라이브러리를 써야 해서
여태 만들어놓은 코드를 사용할 수 없음
